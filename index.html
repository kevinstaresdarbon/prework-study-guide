HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • There are still some semantic tags that appear within the body like "section"
  • HTML uses opening and closing tags for elements. closing tags use the same name as the opening tag but prefixed with /
  • In HTML you should use the h1 to h6 tags in numerical order within each section to create a natural look and flow and the h1 tag should only be used once on a page for its main header
  • Elements can contain additional information stored in the value of its attributes
  • The class attribute is often used to help target the element with some CSS styling
  • The id attribute is typically used to help target the element with Javascript
  • The ul (unordered list) element or ol (ordered list) contains a group of li (list item) elements to arrange them in an automatically bullet pointed list on screen
  • the html element is the root element and can contain a lang attribute of two letter codes that tells the browser the desired language to use; en stands for english
  • The head tag is used to declare metadata and import libraries and styles from external sources
  • The script tag is generally used to import and run some javascript
  • the p tag defines a body of text to be formatted as a single paragraph
  • br can be used as a self closing tag to enter a seperating line between elements
  • Many tags can "self-close", which is to say you can close it with a / just before the end of the element tag instaead of using a seperate closing tag.
  • The title tag that is used in the head section does visibly affect our tab by changing the text within the title of the open tab
  • The body tag is where we write almost all of the visible markup for the page
  • div and span tags help us to control and divide the screen into difeerent functional areas that can be targetted by Javascript and CSS
  • The img tag is used to bring images into our webpage from a particular src (source)
  • The a element defines an anchor, generally used to create a hyperlink to another page on the internet, possibly outside of our current site by including the url in the href attribute
  • header, main and footer are tags used to seperate our elements into specified zones.

GIT

  • GIT is a version control tool that is used widely by development teams
  • GIT and GITHUB makes it a lot easier for developers to work individually and as a collaborative effort upon a code base
  • A reference of git commands can be found here for ease of reference here ---~> GIT CHEAT SHEET
  • The git workflow of branching, editing, pulling and merging is essential knowledge for any software developer
  • The main branch usually contains the code for the live site that is being developed or maintained
  • Its important to branch before making any changes to the code base so that your edited code does not immediately effect or derail the main branch by adding errors or unoptimised features
  • Its a typical convention to name your branch depending on what specifically you're trying to do for the ease of reference of other developers on your team. Usually something like "feature/updated hyperlinks"
  • Try to isolate each branch to only editing a specific feature or resolve a particular issue to help avoid merge conflicts and to make it easier to merge your code or do less work to revert ineffective changes
  • Pull requests are an important part of the workflow as they allow you r code to be reviewed and discussed before the changes are merged into another branch. This helps to ensure quality of code, functionality and reduces the likelyhood of breaking a branch or a build
  • "Git ignore" is an extremely important command when working with npm toolchains as it stops you from committing unecessary code and helps protect your credential files
  • Staging is an important concept to grasp. Once branched you can edit the files on your branch at will but nothing will change with a "git push" or "git commit" if you have not staged the files to change with "git add" first
  • GITHUB is an incredibly useful website that hosts developers repositiories and comes with in-built tools like "Issues", "wiki" and "project" to help document, plan and delegate work that is done on the code base
  • GITHUB also allows for the storage and use of ssh keys to protect and validate your work for authenticating and signing your materials / commits
